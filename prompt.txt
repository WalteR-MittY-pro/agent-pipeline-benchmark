Candidate PR Criteria
To be marked as a "Candidate", a PR MUST satisfy ALL of the following rules:

Rule 1: Maximum File Limit (Strictly Enforced)
The PR MUST NOT modify more than 10 files in total. If the total count of changed files is greater than 10, discard the PR immediately. This ensures we only analyze focused and highly cohesive cross-language changes.

Rule 2: Multi-Language Modification
The PR must contain modified files from AT LEAST TWO different primary programming languages.

Examples of distinct language extensions: .go, .java, .ts/.js, .cj, .kt, .cpp/.h, .py, .rs, .swift.
(Note: Ignore configuration, documentation, or dependency files like .json, .yaml, .md, .xml, pnpm-lock.yaml, go.mod when counting distinct languages).
Rule 3: Test File Inclusion
The PR must contain at least one modified or newly added test file.

Look for common test file naming conventions, such as: _test.go, .test.ts, .spec.ts, Test.java, *Tests.kt, or files under a test/ or tests/ directory.
